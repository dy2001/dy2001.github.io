Gogs

Gogs 的目标是打造一个最简单、最快速和最轻松的方式搭建自助 Git 服务。使用 Go 语言开发使得 Gogs 能够通过独立的二进制分发，并且支持 Go 语言支持的 所有平台，包括 Linux、Mac OS X、Windows 以及 ARM 平台。










以上来自Gogs官网

安装

Gogs是支持docker安装了,简直不要太方便。

下载gogs镜像

docker pull gogs/gogs

创建本地数据目录

mkdir -p /home/dockerVolume/gogs
路径可以自己改到自己习惯存放的地方，如果以后要更换服务器的话只需要把这个文件夹备份了，存到新的服务器上直接docker run就行了。

docker run 运行gogs

docker run -d --name=gogs -p 2003:22 -p 2002:3000 -v /home/dockerVolumes/gogs:/data gogs/gogs

  • -d: 后台方式运行容器
  • -p: 端口映射, 将容器的22端口映射到宿主机的2003端口, 将容器的3000端口映射到宿主机的2002端口
    22是ssh的端口，3000就是网站的端口，然后映射到宿主机的2002盒2003端口，这两个可以改成自己习惯的端口号，如果用的阿里云服务器的话记得要开放端口才能范访问。
  • –name: 指定容器名称
  • -v: 数据卷挂载, 用于将容器和数据分离(注意这里要和上面的数据目录一样)

安装

打开 你的ip:2002,本地的话就是localhost:2002或者127.0.0.1:2002，这里的2002就是上面你和3000绑定的端口。

  • 仓库根目录：将其保留为默认值，/data/git/gogs-repositories因为start.sh已经为您建立了符号链接。
  • 运行系统用户：将其保留为默认值，git因为finalize.sh已经使用名称设置了用户git。
  • 域名：填写Docker容器IP（例如192.168.99.100）。但是，如果要从其他物理机访问Gogs实例，请填写Docker主机的主机名或IP地址。
  • SSH端口：使用Docker容器中公开的端口。例如，您的SSH服务器22在Docker内部监听，但是您通过公开了它2003:22，然后将其2003用于此值。不建议在DockerContainer中使用内置SSH服务器
  • HTTP端口：使用您希望Gogs在Docker容器中侦听的端口。例如，您的Gogs 3000在Docker内部进行侦听，并通过公开了它2002:3000，但仍使用3000该值。
  • 应用程序URL：结合使用域和公开的HTTP端口值（例如http://127.0.0.1:3000/）。
